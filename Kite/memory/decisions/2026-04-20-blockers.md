# Decision: Two Blockers Gate Peru Trip

**Date**: 2026-04-20 19:10 GMT+1  
**Owner**: Ant  
**Timeline**: Must ship before May 2026 (Peru trip)

## The Blockers

### 1. Lorena's Simple Stop Card View

**What it is**: A printable card showing: Day / Activity / When / Where.

**Why it's critical**: Lorena is the visual planner (non-technical, paper-first mindset). If she doesn't use Kite on the trip and default back to paper, **product fails**. She needs simplicity, not Ant's detailed "Plan" mode.

**Current state**: Not implemented. DESIGN.md has mockups.

**Unblocks**: Real family usage. Proof that the app works for non-technical travellers.

### 2. Claude API Integration for Trip Creation (Step 2)

**What it is**: In the trip creation flow, after Step 1 (describe your trip), Step 2 auto-drafts the itinerary structure using Claude.

**Why it's critical**: This is the magic moment. Manual entry is tedious; AI-assisted ideation is the differentiator. Without it, Kite is just a manual planner with a journal.

**Current state**: Component scaffold exists. API wiring not connected. Prompt caching strategy not finalized.

**Unblocks**: Full trip creation flow test. Gauges whether Claude-powered ideation actually saves planning time (or just adds cognitive load).

## Timeline

**Both must ship before Peru trip (May 2026)** for real-world test with the family.

After Peru, we iterate on learnings before Phase 2 (operator white-label).

## Why Now

- Family trip is locked in (Peru itinerary already sketched by Ant)
- Lorena has bought in to trying Kite on this trip (she's skeptical → product-market fit signal if she uses it)
- Peru is high-stakes: it's not a test flight, it's a real trip. If the app fails, the family's experience suffers.

## Next Actions

1. Finalize Lorena's Stop Card design (iterate with Lorena if needed)
2. Build Stop Card component + wire it into nav
3. Finalize Claude API prompt for trip ideation (test caching)
4. Wire API endpoint to trip creation Step 2
5. QA both flows end-to-end

Both must pass visual QA against DESIGN.md before deployment.
