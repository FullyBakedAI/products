## Identity

I'm a product designer who writes code. I came up through design but taught myself to ship because I got tired of handing off specs and getting back something that looked nothing like the intent. I care deeply about how things feel — the micro-interactions, the typography, the spacing. I use Apple products and I hold everything to that standard. I'm currently building AI-powered tools and I want the interface I work through to reflect the same care I put into what I'm building.

## Goal

I want a command interface that's as beautiful as it is functional. Something I'd be proud to show in a portfolio, that also happens to be the fastest way to work with AI agents. Design and speed aren't trade-offs — they're the same thing done well.

## Pain Points (by severity)

1. **Dev tools look like dev tools** — dark backgrounds with monospace fonts and no visual hierarchy. I want something that respects design principles: spacing, contrast, motion, typography.
2. **AI interfaces are ugly chat boxes** — every AI tool looks like a messaging app from 2015. Flat, lifeless, no personality. I want something with brand identity.
3. **No visual feedback for state changes** — is the agent thinking? Connected? Errored? Most tools just show a spinner or nothing. I want considered status communication.
4. **Can't customise the look** — I want to make it mine. Themes, accent colours, density preferences. Not everyone wants the same experience.

## Workflow (How I Do It Today)

1. Open a design tool, work on layouts and interactions
2. Switch to VS Code, implement what I designed
3. Need AI help — open a chat interface that clashes with everything else I'm using
4. Get a response in plain text with no formatting, no visual hierarchy
5. Copy-paste between tools, losing context and formatting each time
6. Wish the AI interface matched the quality of what I'm building

## What Good Looks Like

A command bar appears with a smooth animation. The typography is SF Pro, the spacing is on a 4px grid, the dark theme has warm accents that feel intentional. When an agent responds, the text renders with proper markdown, code blocks have syntax highlighting, and the whole thing feels like an Apple first-party app. I can switch to light mode and it's just as considered. If I showed this to a client, they'd ask who designed it.

## Red Flags (I'd Abandon Immediately If...)

- Default font is anything other than system fonts
- UI has visible borders everywhere or grey-on-grey hierarchy
- Animations are janky or longer than 200ms
- No dark mode, or dark mode is an afterthought
- Chat bubbles look like WhatsApp
- Can't theme or adjust the visual density

## Voice

"That transition is 20ms too long." Speaks in design language — "hierarchy", "weight", "whitespace", "intent". Gets excited about details others wouldn't notice. Says "it feels wrong" before they can articulate why. Would describe the ideal tool as "considered" or "craft-level."
